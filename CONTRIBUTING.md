# Contributing to Islam's Portfolio

Thank you for your interest in contributing to this portfolio website! This is a personal portfolio project, but I'm open to suggestions, bug reports, and improvements.

## ü§ù How to Contribute

### Reporting Issues
If you find any issues with the website (broken links, display problems, accessibility concerns):

1. Check existing issues first
2. Create a new issue with:
   - Clear description of the problem
   - Steps to reproduce
   - Browser/device information
   - Screenshots if applicable

### Suggesting Improvements
I welcome suggestions for:
- Performance optimizations
- Accessibility enhancements
- Mobile experience improvements
- New feature ideas

### Code Contributions
While this is a personal portfolio, I'm open to technical improvements:

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**: Ensure they align with the existing design system
4. **Test thoroughly**: Check responsiveness across devices
5. **Submit a pull request**: Describe your changes clearly

## üé® Design Guidelines

### Color Scheme
- **Primary**: #b97552 (coral accent)
- **Background**: #FBF7F2 (warm light)
- **Text**: #2C3136 (dark charcoal)
- **Secondary**: #f4f0ec (light accent)

### Typography
- **Headings**: Poppins (600 weight)
- **Body**: Segoe UI (400 weight)
- **Navigation**: Open Sans (600 weight)

### Design Principles
- **Clean & Minimal**: Avoid clutter, maintain whitespace
- **Warm & Inviting**: Use the established color palette
- **Professional**: Maintain business-appropriate presentation
- **Accessible**: Ensure WCAG 2.1 AA compliance

## üì± Responsive Requirements

All contributions must maintain responsiveness across:
- **Desktop**: 1200px+ screens
- **Tablet**: 768px-1199px devices
- **Mobile**: 576px-767px phones
- **Small Mobile**: <576px devices

## üîß Technical Guidelines

### Code Standards
- **HTML5**: Use semantic elements
- **CSS3**: Follow BEM-like naming conventions
- **JavaScript**: ES6+ features, clean code practices
- **Accessibility**: ARIA labels, keyboard navigation

### File Organization
- Keep the existing structure intact
- Use descriptive file names
- Maintain the image organization in `/images/`

### Performance
- Optimize images for web
- Minimize external dependencies
- Ensure fast loading times
- Test on various network conditions
